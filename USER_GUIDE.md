# TestCo-pilot 使用指南

智能测试助手 — 基于大模型的测试报告分析和知识库问答工具

---

## 快速开始

### 安装

1. 下载 `TestCo-pilot_0.1.0_x64_en-US.msi`
2. 双击运行安装程序
3. 按照提示完成安装
4. 桌面或开始菜单找到 TestCo-pilot 启动

### 首次启动

1. 阅读并同意使用声明
2. 进入设置页面配置 API Key
3. 开始使用

---

## 功能说明

### 1. AI 对话

**支持的模型服务商：**

| 服务商 | 模型 |
|--------|------|
| 小米 MiMo | V2.5, V2.5-Pro |
| 阿里 通义千问 | Turbo, Plus, Max |
| DeepSeek | V3, R1 |
| 智谱 GLM | Flash, 4, Plus |
| 百度 文心一言 | ERNIE 系列 |
| 字节 豆包 | Lite, Pro |
| 月之暗面 Kimi | V1 系列 |
| OpenAI | GPT-4o, 4o-mini |
| Claude | 3.5 Sonnet, 3 Opus |
| 其他 | 自定义兼容 OpenAI API |

**操作方式：**
- 输入文字按 Enter 发送
- Shift+Enter 换行
- Ctrl+V 粘贴截图
- 拖拽图片到窗口
- 输入框可拖拽调整高度
- 右键对话可重命名/置顶/删除

### 2. 知识库

**支持格式：** TXT、Markdown、PDF、DOCX、XLSX

**使用方法：**
1. 在设置中配置知识库扫描目录
2. 将文档放入该目录
3. 点击"扫描"按钮导入
4. 系统自动分块、向量化、分类

**智能分类：**
- 刷写（Flash、OTA）
- 诊断（UDS、DTC）
- 通讯（CAN、LIN、Ethernet）
- 网络管理（NM、AUTOSAR）
- 路由（Gateway）
- 其他

### 3. 分析报告

**支持格式：** XML 报告、ZIP 压缩包

**使用方法：**
1. 点击"上传报告"
2. 选择 XML 或 ZIP 文件
3. 系统自动解析失败用例
4. 查看详细分析结果

### 4. 截图分析

- Ctrl+V 粘贴剪贴板截图
- 拖拽图片文件到对话框
- 点击图片图标选择文件

## 设置说明

### API 配置

1. 选择模型服务商
2. 填写 API Key
3. 点击"测试连接"验证
4. 点击"保存配置"

### 本地模型

1. 下载 GGUF 格式模型文件
2. 在设置中选择文件路径
3. 点击"加载模型"
4. 在对话界面切换到本地模型

**推荐模型：**
- Qwen2.5-7B / 14B
- DeepSeek-R1-Distill
- LLaMA-3-8B
- Mistral-7B

### 存储路径

- **知识库存储路径**：索引数据存储位置
- **知识库扫描目录**：文档扫描目录
- **分析报告路径**：报告保存位置

## 常见问题

**Q: API 连接失败？**
A: 检查 API Key 是否正确，网络是否正常。

**Q: 知识库搜索无结果？**
A: 确保已扫描目录，文档格式正确。

**Q: 本地模型加载失败？**
A: 确认 GGUF 文件完整，内存充足。

**Q: 如何更新？**
A: 设置页面点击"检查更新"，或重新下载安装包。
